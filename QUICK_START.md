# クイックスタートガイド

## 📋 必要なもの

- Python 3.10.x
- インターネット接続（初回セットアップのみ）

---

## 🚀 5分で始める

### 1️⃣ 依存パッケージをインストール

```bash
pip install -r requirements.txt
```

### 2️⃣ アプリを起動

```bash
streamlit run app.py
```

自動的にブラウザが開きます！

---

## 🎯 最初の試算

1. **左サイドバー**で以下を設定：
   - 機種選択（デフォルトで3機種が選択済み）
   - リードタイム: 7日（稼働日）
   - 欠品許容率: 10%

2. **メイン画面**で結果を確認：
   - タブ「安全在庫①（理論値）」: 理論式による計算結果
   - タブ「安全在庫②（実測値）」: 実測データによる計算結果
   - タブ「比較」: 両者の比較表とグラフ

3. **グラフとデータをエクスポート**：
   - 「📥 分布データをCSV保存」ボタン → 分析データを保存
   - **PNG保存**: Plotlyグラフ右上のカメラアイコンをクリック、またはブラウザのスクリーンショット機能を使用

---

## 🎨 パラメータで遊んでみよう

### リードタイムを変更
- **稼働日7日** → 安全在庫はどう変わる？
- **カレンダー7日** → 稼働日換算でどうなる？
- **最大60日まで設定可能** → 海外製品の生産＋輸送リードタイム（約2か月）にも対応

### 欠品許容率を変更
- **5%** → 安全在庫が増える（サービスレベル向上）
- **20%** → 安全在庫が減る（在庫コスト削減）

### 標準偏差計算方法を変更
- **区間標準偏差の平均（推奨）** → より現実的な変動を反映
- **全期間標準偏差** → 全体の傾向を反映

---

## 🔍 結果の見方

### 安全在庫①（理論値）
- **グラフ**: リードタイム期間の需要分布（理論的な正規分布）
- **赤い縦線**: 安全在庫の位置
- **意味**: この在庫量を持てば、設定した欠品許容率でカバー可能

### 安全在庫②（実測値）
- **グラフ**: 計画値と実績値の差分分布（実際のデータ）
- **灰色の縦線**: 差分=0（計画と実績が一致）
- **赤い縦線**: 安全在庫の位置
- **意味**: 計画精度のばらつきを考慮した安全在庫

### 比較タブ
- **理論値 vs 実測値**: どちらがより現実的か？
- **差分・差分率**: 両者の乖離を定量化

## 💡 Tips

### 複数機種を一度に分析
サイドバーで複数の機種を選択すると、まとめて分析できます。

### データを変更したい
1. `data/日次計画.csv` と `data/日次実績.csv` を編集
2. アプリを再起動（ブラウザでF5キー）

### 計算が遅い場合
キャッシュが効いているので、2回目以降は高速です。
パラメータを変更すると自動的に再計算されます。

## ❓ よくある質問

**Q: グラフが表示されない**
A: ブラウザを再読み込み（F5キー）してください。

**Q: PNG保存ボタンが表示されない**
A: 現在、PNG保存機能は一時的に無効化されています。Plotlyグラフ右上のカメラアイコンをクリックするか、ブラウザのスクリーンショット機能（Win + Shift + S）を使用してください。

**Q: 機種が選択肢に出ない**
A: `data/日次計画.csv` にその機種のデータがあるか確認してください。
